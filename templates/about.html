Sobre o projeto

Trata-se de um sistema de gerenciamento de produtos fictícios, em que o usuário pode interagir com o banco de dados MySQL de forma intuitiva por meio da aplicação Web. As páginas Web são construídas com templates Jinja para permitir interação com Python e tem a formatação ajustada via Bootstrap. O usuário pode realizar operações típicas (compra, venda, movimentação) com os produtos e visualizar os dados dos bancos em tabela. As tabelas, que contém entidades relacionadas entre si, são as seguintes:

Tabela produtos: contém o ID, nome, custo (de fabricação/aquisição) e preço (de venda ou "capitalização líquida") de cada produto.
Tabela lotes: contém o ID de cada lote, bem como o ID do container em que está localizado, o ID do produto que o compõe e quantidade do produto em questão.
Tabela containers: contém o tipo, a localização e o ID de cada container.
Tabela transações: contém o ID de cada transação, o ID do produto sendo manipulado, o ID do container onde ocorre a transação, a quantidade, o horário que a transação ocorreu e o tipo de transação (compra, venda, retirada ou recebimento).
